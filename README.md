# Подпольщик Билл — Telegram-бот Сопротивления

Бот для Telegram-сообщества «ополченцев галактики Diaverse». Ведёт учёт активностей участников: шаги (10 000 в день) и зарядка. Персонаж бота — старый космический пират-подпольщик **Подпольщик Билл**. Тон — саркастичный, ироничный, революционный, но добрый в душе.

---

## 1. Как работает бот

- Участники публикуют скриншот шагомера с текстом `+1` в топик **#шаги** → бот засчитывает день шагов.
- Участники публикуют видео зарядки с текстом `+1` в топик **#зарядка** → бот засчитывает день зарядки.
- Одна запись в день на каждую активность независимо друг от друга.
- Если кто-то жульничает — любой активный участник пишет `/report` в ответ на подозрительное сообщение. Сообщество голосует. 10 голосов за 24 часа — нарушитель отправляется в карцер до конца месяца.
- Команды `/stats` и `/top` показывают статистику бойцов.

## 2. Настройка Supabase

### 2.1 Создать проект

1. Зайти на [supabase.com](https://supabase.com) и зарегистрироваться.
2. Нажать **New project**, выбрать организацию, задать имя проекта и пароль базы данных.
3. Дождаться создания проекта (около минуты).

### 2.2 Запустить schema.sql

1. В боковом меню выбрать **SQL Editor**.
2. Нажать **New query**.
3. Скопировать содержимое файла `schema.sql` из этого репозитория и вставить в редактор.
4. Нажать **Run** (или `Ctrl+Enter`).
5. Убедиться что все таблицы создались: в меню **Table Editor** должны появиться `users`, `activities`, `jails`, `reports`, `report_votes`, `admins`.

### 2.3 Получить ключи

1. В боковом меню перейти в **Project Settings → API**.
2. Скопировать:
   - **Project URL** → это `SUPABASE_URL` (вида `https://xxxxxxxxxxxx.supabase.co`)
   - **anon / public** key → это `SUPABASE_KEY`

## 3. Настройка Telegram

### 3.1 Создать бота

1. Написать [@BotFather](https://t.me/BotFather) в Telegram.
2. Отправить команду `/newbot`.
3. Следовать инструкциям: придумать имя и username для бота.
4. Скопировать **токен** вида `123456789:AAF...` → это `BOT_TOKEN`.

### 3.2 Узнать GROUP_ID

**Способ 1 — через getUpdates:**

1. Добавить бота в супергруппу и дать ему права администратора.
2. Написать любое сообщение в группу.
3. Открыть в браузере:
   ```
   https://api.telegram.org/bot<BOT_TOKEN>/getUpdates
   ```
4. В ответе найти поле `"chat"` → `"id"`. Для супергруппы это отрицательное число вида `-1001234567890` → это `GROUP_ID`.

**Способ 2 — через @userinfobot:**

1. Добавить [@userinfobot](https://t.me/userinfobot) в группу.
2. Написать в группе что угодно и переслать боту — он ответит с ID чата.
3. Удалить @userinfobot из группы после получения ID.

### 3.3 Узнать STEPS_THREAD_ID и EXERCISE_THREAD_ID

Эти ID — идентификаторы топиков (тем) в супергруппе с включёнными темами.

**Способ — через getUpdates:**

1. Убедиться что в группе включены темы (**Topics**): Настройки группы → Topics → включить.
2. Создать топики **#шаги** и **#зарядка** если их ещё нет.
3. Отправить любое сообщение в топик **#шаги**.
4. Открыть:
   ```
   https://api.telegram.org/bot<BOT_TOKEN>/getUpdates
   ```
5. В ответе найти `"message_thread_id"` — это и есть `STEPS_THREAD_ID`.
6. Повторить для топика **#зарядка** → получить `EXERCISE_THREAD_ID`.

> **Подсказка:** Если `getUpdates` возвращает пустой список, сначала удалите старые обновления:
> `https://api.telegram.org/bot<BOT_TOKEN>/getUpdates?offset=-1`

### 3.4 Узнать OWNER_ID

1. Написать боту [@userinfobot](https://t.me/userinfobot) напрямую (не в группу).
2. Он ответит вашим Telegram user ID → это `OWNER_ID`.

## 5. Локальный запуск

### Требования

- Python 3.11+
- pip

### Установка и запуск

```bash
# Клонировать репозиторий
git clone <url>
cd diaverse-bot

# Установить зависимости
pip install -r requirements.txt

# Запустить бота
python bot.py
```

В консоли должно появиться сообщение `Подпольщик Билл выходит на связь...` — бот работает.

Для остановки нажать `Ctrl+C`.

## 6. Деплой на Railway

[Railway](https://railway.app) — бесплатный (в разумных пределах) хостинг для Python-приложений.

### Шаги

1. Зарегистрироваться на [railway.app](https://railway.app) (удобнее через GitHub).
2. Нажать **New Project → Deploy from GitHub repo**.
3. Выбрать репозиторий с ботом. Если репозиторий не виден — нажать **Configure GitHub App** и дать Railway доступ.
4. Railway автоматически определит Python-проект. Нажать **Deploy**.
5. Перейти в раздел **Variables** проекта и добавить все переменные из `.env`:

   | Переменная | Значение |
   |---|---|
   | `BOT_TOKEN` | токен от @BotFather |
   | `SUPABASE_URL` | URL из Supabase |
   | `SUPABASE_KEY` | anon key из Supabase |
   | `OWNER_ID` | ваш Telegram user ID |
   | `GROUP_ID` | ID супергруппы |
   | `STEPS_THREAD_ID` | ID топика шагов |
   | `EXERCISE_THREAD_ID` | ID топика зарядки |

6. В разделе **Settings → Deploy** убедиться что **Start Command** задан как:
   ```
   python bot.py
   ```
7. Нажать **Redeploy** если деплой уже запущен, или дождаться автоматического деплоя.
8. В разделе **Logs** убедиться что бот запустился без ошибок.

> **Важно:** Railway может усыплять сервис при отсутствии активности на бесплатном плане. Для стабильной работы бота рекомендуется план Hobby ($5/мес) или настройка keep-alive.

## 7. Первый запуск: настройка в группе

1. **Добавить бота в супергруппу.**
2. **Дать боту права администратора** (иначе он не сможет читать сообщения в топиках):
   - Настройки группы → Администраторы → Добавить администратора → выбрать бота.
   - Достаточно прав: **чтение сообщений**, **отправка сообщений**, **редактирование сообщений**.
3. **Убедиться что OWNER_ID указан верно** — только этот пользователь может назначать других администраторов командой `/addadmin`.
4. Написать `/help` в группе — бот должен ответить инструкцией.
5. Протестировать: отправить фото с текстом `+1` в топик шагов — бот должен ответить reply.

## 8. Команды бота

### Для всех участников

| Команда | Описание |
|---|---|
| `/help` | Инструкция по работе с ботом |
| `/start` | То же что `/help` |
| `/stats` | Своя статистика за текущий месяц |
| `/stats @username` | Статистика другого бойца |
| `/stats` (reply) | Статистика того, на чьё сообщение отвечаешь |
| `/top` | Таблица всех участников за текущий месяц |
| `/report` (reply) | Подать жалобу на сообщение (только в тот же день) |

### Для администраторов и владельца

| Команда | Описание |
|---|---|
| `/pardon @username` | Досрочно снять карцер с бойца |
| `/reset` | Снять все карцеры и объявить новый месяц |

### Только для владельца (OWNER_ID)

| Команда | Описание |
|---|---|
| `/addadmin @username` | Назначить бойца администратором |

## Технические детали

- Все даты считаются по **московскому времени** (Europe/Moscow).
- Бот работает в режиме **polling** (не webhook) — проще в деплое.
- Голосование по жалобе автоматически закрывается через **24 часа** через встроенную JobQueue.
- Карцер действует **до конца текущего месяца** (последний день месяца по МСК).
