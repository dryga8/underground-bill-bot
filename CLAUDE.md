# Diaverse Resistance Bot — Подпольщик Билл

## Обзор проекта

Telegram-бот для сообщества "ополченцев галактики Diaverse" (~100 человек).
Бот ведёт учёт активностей участников: шаги (10 000 в день) и зарядка.
Персонаж бота — старый космический пират-подпольщик по имени **Подпольщик Билл**.
Язык общения — только русский. Тон — саркастичный, ироничный, революционный, но добрый в душе.

## Стек

- **Python 3.11+**
- **python-telegram-bot v20.7** (async)
- **Supabase** (PostgreSQL через supabase-py)
- **pytz** для работы с МСК временем
- **python-dotenv** для переменных окружения

## База данных (schema.sql)

```sql
-- Пользователи
CREATE TABLE users (
    user_id BIGINT PRIMARY KEY,
    username TEXT,
    first_name TEXT,
    last_name TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Активности
CREATE TABLE activities (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT REFERENCES users(user_id),
    activity_type TEXT NOT NULL CHECK (activity_type IN ('steps', 'exercise')),
    activity_date DATE NOT NULL,
    month INTEGER NOT NULL,
    year INTEGER NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(user_id, activity_type, activity_date)
);

-- Карцер
CREATE TABLE jails (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT REFERENCES users(user_id),
    jailed_at TIMESTAMPTZ DEFAULT NOW(),
    jailed_until DATE NOT NULL,
    active BOOLEAN DEFAULT TRUE
);

-- Жалобы
CREATE TABLE reports (
    id BIGSERIAL PRIMARY KEY,
    reporter_id BIGINT REFERENCES users(user_id),
    reported_user_id BIGINT REFERENCES users(user_id),
    chat_id BIGINT NOT NULL,
    message_id BIGINT NOT NULL,
    vote_message_id BIGINT,
    thread_id INTEGER,
    status TEXT DEFAULT 'open' CHECK (status IN ('open', 'jailed', 'cleared')),
    yes_votes INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    expires_at TIMESTAMPTZ NOT NULL
);

-- Голоса
CREATE TABLE report_votes (
    id BIGSERIAL PRIMARY KEY,
    report_id BIGINT REFERENCES reports(id),
    voter_id BIGINT REFERENCES users(user_id),
    voted_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(report_id, voter_id)
);

-- Админы
CREATE TABLE admins (
    user_id BIGINT PRIMARY KEY REFERENCES users(user_id),
    added_by BIGINT,
    added_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_activities_user_month ON activities(user_id, month, year);
CREATE INDEX idx_jails_active ON jails(user_id, active);
CREATE INDEX idx_reports_status ON reports(status);
```

## Логика работы

### Приём активности (handlers/activity.py)

Бот слушает сообщения в супергруппе. Условия для засчитывания:

**Шаги** (топик STEPS_THREAD_ID):
- Сообщение содержит ФОТО (скриншот)
- Текст/подпись содержит число >= 10 000 (пробелы внутри числа нормализуются: "10 500" → 10500)
- Если числа нет → STEPS_NOT_RECOGNIZED; если число < 10 000 → TOO_FEW_STEPS
- После записи начисляется XP (steps_count // 500) и обновляется total_steps

**Зарядка** (топик EXERCISE_THREAD_ID):
- Сообщение содержит ВИДЕО
- Текст сообщения содержит "+1" (регистронезависимо)

**Алгоритм для шагов:**
1. Извлечь максимальное число из текста/caption
2. Если нет числа → STEPS_NOT_RECOGNIZED
3. Если число < 10 000 → TOO_FEW_STEPS
4. Upsert пользователя в БД
5. Проверить карцер → JAILED_TRY
6. Проверить дубль сегодня → ALREADY_SUBMITTED_STEPS
7. Записать в activities (record_steps с steps_count)
8. Начислить XP в таблицу xp, обновить total_steps в таблице total_steps
9. Ответить фразой STEPS_ACCEPTED с числом шагов

**Важно:**
- Дата определяется по МСК (Europe/Moscow)
- Одна запись в день на каждую активность независимо друг от друга
- Бот отвечает REPLY на исходное сообщение пользователя

### Жалоба (handlers/report.py)

**Команда `/report`** — только в ответ (reply) на сообщение:

1. Проверить что команда — это reply на чьё-то сообщение
2. Проверить что жалоба подаётся в тот же день (МСК) что и исходное сообщение
3. Проверить что reporter активен в текущем месяце (есть хоть одна запись в activities за текущий месяц)
4. Проверить что на это сообщение нет открытой жалобы
5. Создать запись в reports (expires_at = now + 24h)
6. Отправить сообщение с inline-кнопками в ту же ветку:
   - Кнопка "✅ Да, обманул" → callback `vote:yes:{report_id}`
   - Кнопка "❌ Нет, всё чисто" → callback `vote:no:{report_id}`
7. Запланировать job через 24 часа → `close_expired_report(report_id)`

**Голосование (callback `vote:yes` / `vote:no`):**

Только кнопка "Да" влияет на исход. "Нет" — просто для UX.

При нажатии "Да":
1. Проверить что voter активен в текущем месяце
2. Проверить что voter ещё не голосовал по этой жалобе
3. Добавить голос, обновить счётчик
4. Если yes_votes >= 10:
   - Посадить в карцер до последнего дня текущего месяца
   - Закрыть жалобу (status = 'jailed')
   - Отредактировать сообщение с голосованием → показать результат
   - Отправить объявление фразой из JAIL_VERDICT
5. Если < 10: ответить callback answer с текущим счётчиком (тихо)

При нажатии "Нет":
- Ответить callback answer "Твой голос учтён" (тихо, без последствий)

**Job закрытия через 24ч:**
1. Проверить статус жалобы — если уже не 'open': ничего не делать
2. Закрыть жалобу (status = 'cleared')
3. Отредактировать сообщение голосования → показать результат
4. Отправить объявление фразой из VOTE_EXPIRED

### Статистика (handlers/stats.py)

**`/stats`** — показывает досье на бойца:
- Если команда — reply на чьё-то сообщение → статистика того человека
- Если `/stats @username` → ищем по username в БД
- Если просто `/stats` → своя статистика

Формат ответа:
```
🗂 Досье на бойца: Имя Фамилия

📅 Май 2025:
  🚶 Шагает: 23 дня
  ⚡ Заряжается: 15 дней
```

**`/top`** — таблица всех участников текущего месяца, отсортированная по алфавиту имён:

```
🏆 Таблица героев Сопротивления
📅 Май 2025

Имя                  🚶 Шагает     ⚡ Заряжается
Иван Петров          23 дня        15 дней
Маша Сидорова        20 дней       18 дней
```

Использовать моноширинный шрифт (`<pre>` тег) для выравнивания колонок.
parse_mode=HTML.

### Приветствие новых участников (handlers/welcome.py)

При вступлении нового пользователя в группу:
1. Бот отправляет случайное приветствие из `WELCOME_MESSAGES` с тегом пользователя (`mention_html()`)
2. Сообщение отправляется в NEWS_THREAD_ID (если задан) или в основной чат
3. Через 3 минуты сообщение автоматически удаляется через `job_queue.run_once`
4. Боты игнорируются (is_bot check)

### Админ-команды (handlers/admin.py)

**`/addadmin @username`** — только OWNER_ID:
- Найти пользователя по username в таблице users
- Добавить в таблицу admins

**`/pardon @username`** — только админы и OWNER_ID:
- Найти пользователя по username
- Деактивировать все активные записи в jails (active = False)
- Ответить фразой из JAIL_LIFTED

**`/reset`** — только админы и OWNER_ID:
- Деактивировать ВСЕ активные записи в jails (active = False)
- Активности НЕ удаляются (история сохраняется)
- Ответить объявлением о начале нового месяца

## Важные детали реализации

1. **Timezone**: все проверки "сегодня" и "этот месяц" делаются по МСК (Europe/Moscow)
2. **Reply**: бот ВСЕГДА отвечает reply на сообщение пользователя (message.reply_text)
3. **parse_mode**: использовать HTML везде где нужно форматирование
4. **JobQueue**: для закрытия голосования через 24ч использовать context.job_queue.run_once()
5. **Карцер**: jailed_until = последний день текущего месяца по МСК
6. **Проверка активности для жалоб**: reporter должен иметь хотя бы одну запись в activities за текущий месяц/год (МСК)
7. **Один репорт на сообщение**: нельзя подать жалобу на сообщение если по нему уже открыто голосование
8. **Жалоба только в тот же день**: message_date (из Telegram) должна совпадать с текущей датой по МСК
9. **Лидерборд**: пустые строки (0 дней) тоже показываем если пользователь есть в таблице activities за месяц. Сортировка по алфавиту display_name.
10. **display_name**: first_name + last_name если есть, иначе username (без @), иначе "Неизвестный боец"
11. **XP**: начисляется за шаги = steps_count // 500. Хранится в таблице `xp` (user_id, total_xp). Upsert через select+update или upsert.
12. **total_steps**: накопительные шаги за всё время. Хранится в таблице `total_steps` (user_id, all_time_steps). Upsert аналогично.
